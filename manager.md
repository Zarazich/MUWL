# manager — управление зашифрованными чатами через Yandex Mail

Класс `manager` обеспечивает полный цикл работы с защищёнными сообщениями: создание чатов, привязку почтовых ящиков, отправку и получение зашифрованных писем, автоматическую сортировку по чатам на основе проверки маршрута (`route`). Все данные сохраняются в JSON-конфиге.

---

## 📁 Конфигурационный файл

При инициализации указывается путь к JSON-файлу. Если файл отсутствует или повреждён, создаётся новый со структурой:

```json
{
  "hash": null,
  "chats": [],
  "emails": [],
  "name": "USER"
}
```

### Поля конфига

| Поле     | Тип                          | Описание |
|----------|------------------------------|----------|
| `hash`   | string / null                | Хеш мастер-пароля (алгоритм из `cryptohelper.hash_password`) |
| `name`   | string                       | Имя пользователя (отображается при создании чата) |
| `chats`  | list of chat objects         | Список чатов (см. структуру ниже) |
| `emails` | list of email account objects| Список привязанных почтовых ящиков |

### Структура объекта чата (`chats`)

```json
{
  "name": "Название чата",
  "routekey": "строковый ключ для шифрования индексов",
  "route": "строковый адрес (идентификатор) чата",
  "enckey": "зашифрованный мастер-паролем ключ для сообщений чата",
  "emails": ["list@example.com", "friend@yandex.ru"],
  "messages": [["зашифрованное_сообщение", 0], ["...", 1]],
  "unreaden": false
}
```

- `messages` — список кортежей `(encrypted_message, direction)`, где `direction=0` — входящее, `1` — исходящее.
- `unreaden` — флаг наличия непрочитанных сообщений.

### Структура объекта email-аккаунта (`emails`)

```json
{
  "email": "my@yandex.ru",
  "apikey": "пароль_приложения",
  "lastuid": null
}
```

- `lastuid` — последний полученный UID письма (используется для инкрементальной загрузки).

## 🧩 Методы класса

### `__init__(self, path_to_conf)`

- **Параметры:** `path_to_conf` — путь к JSON-файлу конфигурации.
- **Действие:** Загружает или создаёт конфиг, устанавливает флаг `self.ready`.

---

### `setup(self, password, name)`

Устанавливает мастер-пароль и имя пользователя. Работает только если `hash` ещё не задан.

- **Параметры:**
  - `password` — мастер-пароль (строка).
  - `name` — имя пользователя (строка).
- **Результат:** Сохраняет хеш пароля и имя в конфиг.

---

### `setup_email(self, email, apikey)`

Добавляет почтовый ящик для приёма/отправки.

- **Параметры:**
  - `email` — адрес ящика.
  - `apikey` — пароль приложения Яндекса.
- **Возвращает:** `True` если добавлен, `False` если уже существует.

---

### `create_chat(self, password, name)`

Создаёт новый чат (требуется мастер-пароль).

- **Параметры:**
  - `password` — мастер-пароль для подтверждения.
  - `name` — название чата (должно быть уникальным).
- **Возвращает:** кортеж `(enckey, routekey, route, user_name)` — ключ шифрования сообщений, ключ маршрута, адрес чата и имя пользователя. Эти данные нужно передать собеседнику.

---

### `create_key(self) -> str`

Генерирует случайный строковый ключ (10 символов из букв+цифр).

- **Используется внутри** для `routekey`, `route` и `enckey`.

---

### `save_config(self)`

Сохраняет текущее состояние `self.data` в JSON-файл. Вызывается автоматически после изменений.

---

### `receive(self)`

Основной метод приёма сообщений.

1. Перебирает все привязанные email-аккаунты.
2. Для каждого вызывает `api_helper.receive` с учётом `lastuid`.
3. Обновляет `lastuid` и помечает ящик как `unreaden`.
4. Для каждого полученного письма (содержит зашифрованное сообщение и email отправителя) проверяет все существующие чаты через `validate_route(routekey, route, message)`.
5. Если маршрут совпадает:
   - Добавляет сообщение в `messages` чата с направлением `0` (входящее).
   - Добавляет email отправителя в список `emails` чата (если ещё не было).
   - Устанавливает `unreaden = True`.
6. Сохраняет конфиг.

---

### `update(self)`

- **Возвращает:** список названий чатов, в которых есть непрочитанные сообщения (`unreaden == True`).

---

### `get_messages_from_chat(self, name_of_chat, password)`

Расшифровывает и возвращает все сообщения указанного чата.

- **Параметры:**
  - `name_of_chat` — название чата.
  - `password` — мастер-пароль.
- **Действие:** Проверяет пароль, находит чат, расшифровывает `enckey`, затем расшифровывает каждое сообщение с помощью `decrypt_message(routekey, enc_key, encrypted_message)`.
- **Возвращает:** кортеж `(messages_list, chat_name)` или `None` при ошибке.  
  `messages_list` — список расшифрованных текстов (сохраняется порядок исходных сообщений).  
  После вызова флаг `unreaden` сбрасывается в `False`.

---

### `send_message(self, name_of_chat, message, email_to, email_from, password)`

Отправляет зашифрованное сообщение в указанный чат.

- **Параметры:**
  - `name_of_chat` — название чата.
  - `message` — текст сообщения (строка).
  - `email_to` — email получателя.
  - `email_from` — ваш email (должен быть привязан через `setup_email`).
  - `password` — мастер-пароль.
- **Действие:** Проверяет пароль, находит чат и email-аккаунт, расшифровывает `enckey`, шифрует сообщение через `encrypt_message(enc_key, routekey, route, message)`, отправляет через `api_helper.send` с темой `"MESSAGE"`.
- **Результат:** Сохраняет зашифрованное сообщение в `messages` чата с направлением `1` (исходящее). Сохраняет конфиг.

---

## 📦 Внешние зависимости

Модули, которые должны быть импортированы (в коде не показаны полностью, но используются):

- `api_helper` — функции `receive` и `send` для работы с IMAP/SMTP Яндекса.
- `cryptohelper` — функции:
  - `hash_password` — получение хеша пароля.
  - `encrypt` / `decrypt` — шифрование/дешифрование ключей мастер-паролем.
  - `encrypt_message` / `decrypt_message` — шифрование/дешифрование сообщений для чата.
  - `validate_route` — проверка маршрута.

---

## ⚠️ Важные замечания

- Мастер-пароль **не хранится** в открытом виде, только его хеш.
- Ключи `routekey` и `route` должны быть переданы собеседнику по защищённому каналу (например, лично или через другой мессенджер).
- Для работы с Яндекс.Почтой необходимо использовать **пароль приложения**, а не основной пароль.
- Письма ищутся по теме `"MESSAGE"` — другие письма игнорируются.
- Автоматически обрабатывается длинная строка base64 (разбивка на 76 символов не нарушает данные).

## 🧪 Пример использования

```python
mgr = manager("config.json")

# Первоначальная настройка
mgr.setup("masterpassword", "Alice")

# Добавление своего ящика
mgr.setup_email("alice@yandex.ru", "app_password")

# Создание чата с Бобом
enckey, routekey, route, my_name = mgr.create_chat("masterpassword", "Bob")

# Передать Бобу: enckey, routekey, route, свой email "alice@yandex.ru"

# Отправка сообщения
mgr.send_message("Bob", "Привет!", "bob@yandex.ru", "alice@yandex.ru", "masterpassword")

# Периодический приём
mgr.receive()
new_chats = mgr.update()   # ['Bob']

# Чтение сообщений
msgs, chat_name = mgr.get_messages_from_chat("Bob", "masterpassword")
print(msgs)  # ['Привет!']
```
