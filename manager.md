# 📘 manager — управление зашифрованными чатами через Yandex Mail

Класс `manager` обеспечивает полный жизненный цикл защищённого обмена сообщениями: создание чатов, привязку почтовых ящиков, отправку и получение зашифрованных писем, автоматическую сортировку по чатам на основе проверки маршрута (`route`). Вся конфигурация хранится в JSON-файле.

---

## ⚙️ Инициализация

```python
mgr = manager(path_to_conf)
```

- **`path_to_conf`** — путь к JSON-файлу конфигурации.
- Если файл не существует или не может быть прочитан, создаётся новый со структурой по умолчанию:
  ```json
  {
    "hash": null,
    "chats": [],
    "emails": [],
    "name": "USER"
  }
  ```
- Атрибут `self.ready` становится `True` только после успешной загрузки или создания конфига.

---

## 🗂️ Структура конфигурационного файла

### Корневой объект

| Поле     | Тип                          | Описание |
|----------|------------------------------|----------|
| `hash`   | string / null                | Хеш мастер-пароля (алгоритм `hash_password` из `cryptohelper`) |
| `name`   | string                       | Имя текущего пользователя (используется при создании чата) |
| `chats`  | list of `Chat`               | Список чатов |
| `emails` | list of `EmailAccount`       | Список почтовых аккаунтов, с которых ведётся приём/отправка |

### Объект `Chat`

```json
{
  "name": "Название чата",
  "routekey": "ключ для шифрования индексов (route)",
  "route": "уникальный идентификатор чата",
  "enckey": "зашифрованный мастер-паролем ключ для сообщений чата",
  "emails": ["alice@yandex.ru", "bob@example.com"],
  "messages": [["зашифрованное_сообщение", 0], ["...", 1]],
  "unreaden": false
}
```

- **`routekey`** и **`route`** — пара, используемая в `validate_route` для определения принадлежности письма чату.
- **`enckey`** — хранит ключ шифрования сообщений, зашифрованный мастер-паролем (через `cryptohelper.encrypt`). Расшифровывается при чтении чата.
- **`emails`** — список email-адресов участников чата (автоматически пополняется при получении новых писем).
- **`messages`** — список кортежей `(encrypted_message, direction)`:
  - `encrypted_message` — строка base64, результат `encrypt_message`.
  - `direction` — `0` для входящих, `1` для исходящих сообщений.
- **`unreaden`** — флаг наличия непрочитанных сообщений в чате.

### Объект `EmailAccount`

```json
{
  "email": "my@yandex.ru",
  "apikey": "пароль_приложения",
  "lastuid": null
}
```

- **`lastuid`** — последний обработанный UID письма (используется для инкрементального получения писем с темой "MESSAGE").
- Поле **`unreaden`** не используется для email-аккаунтов (в отличие от старой документации).

## 🧩 Методы класса

### `setup(password, name)`

Устанавливает мастер-пароль и имя пользователя. Работает **только один раз**, если `self.data["hash"]` ещё пуст.

- **Параметры:**
  - `password` — мастер-пароль (строка).
  - `name` — отображаемое имя пользователя.

### `setup_email(email, apikey)`

Добавляет почтовый аккаунт для приёма/отправки.

- **Параметры:**
  - `email` — адрес ящика.
  - `apikey` — пароль приложения Яндекса.
- **Возвращает:** `True` если добавлен, `False` если такой email уже существует.

### `create_chat(password, name)`

Создаёт новый чат. Требует подтверждения мастер-пароля.

- **Параметры:**
  - `password` — мастер-пароль.
  - `name` — уникальное название чата.
- **Возвращает:** кортеж `(enckey, routekey, route, self.data["name"])`, где
  - `enckey` — ключ шифрования сообщений (нужно передать собеседнику в защищённом канале),
  - `routekey` — ключ для проверки маршрута,
  - `route` — идентификатор чата,
  - последний элемент — имя текущего пользователя (из конфига).
- Если пароль неверен или имя чата уже существует — возвращает `None` (на самом деле в коде просто `return`, что эквивалентно `None`).

### `create_key() -> str`

Вспомогательный метод: генерирует случайную строку из 10 символов (буквы + цифры). Используется для создания `routekey`, `route` и `enckey`.

### `save_config()`

Сохраняет текущее состояние `self.data` в JSON-файл. Вызывается автоматически после любых изменений, если `self.ready == True`.

### `receive()`

Основной метод приёма сообщений.

1. Перебирает все email-аккаунты из `self.data["emails"]`.
2. Для каждого вызывает `api_helper.receive(..., latest_uid=i["lastuid"])`, получая список кортежей `(body, from_email)` и новый `lastuid`.
3. Обновляет `lastuid` в конфиге и **сразу сохраняет конфиг** (чтобы избежать повторной загрузки тех же писем при сбое).
4. Для каждого полученного письма проверяет все существующие чаты через `cryptohelper.validate_route(routekey, route, body)`.
5. Если маршрут совпадает:
   - Добавляет сообщение в список `messages` чата как `(body, 0)` (входящее), если такого сообщения ещё нет (проверка `(message[0], 0) not in messages`).
   - Добавляет email отправителя в список `emails` чата (если его там ещё нет).
   - Устанавливает `unreaden = True` для чата.
   - Выводит отладочную информацию.
6. Все ошибки при получении/обработке конкретного аккаунта перехватываются и выводятся на печать (но не прерывают работу).
7. После обработки всех аккаунтов снова сохраняет конфиг.

### `update() -> list`

- **Возвращает:** список названий чатов, у которых поле `unreaden` равно `True`.

### `get_messages_from_chat(name_of_chat, password)`

Возвращает расшифрованные сообщения указанного чата и сбрасывает флаг `unreaden`.

- **Параметры:**
  - `name_of_chat` — название чата.
  - `password` — мастер-пароль.
- **Действие:**
  - Проверяет пароль.
  - Находит чат по имени.
  - Расшифровывает `enckey` через `cryptohelper.decrypt(password, chat["enckey"])` (получается ключ сообщений `enc_key`).
  - Для каждого сообщения в `chat["messages"]` вызывает `decrypt_message(routekey, enc_key, encrypted_message)`.
  - Устанавливает `chat["unreaden"] = False` и сохраняет конфиг.
- **Возвращает:** кортеж `(список_расшифрованных_текстов, имя_чата)` или `None` при ошибке (неверный пароль, чат не найден, ошибка расшифровки).

### `send_message(name_of_chat, message, email_to, email_from, password)`

Отправляет одно сообщение в указанный чат.

- **Параметры:**
  - `name_of_chat` — название чата.
  - `message` — текст сообщения (строка).
  - `email_to` — email получателя.
  - `email_from` — ваш email (должен быть предварительно добавлен через `setup_email`).
  - `password` — мастер-пароль.
- **Действие:**
  - Проверяет пароль.
  - Находит чат и email-аккаунт отправителя.
  - Расшифровывает `enckey` через `decrypt(password, chat["enckey"])` → `enc_key`.
  - Шифрует сообщение через `encrypt_message(enc_key, routekey, route, message)`.
  - Вызывает `api_helper.send(email_from, email_to, apikey, encmessage)`.
  - Добавляет кортеж `(encmessage, 1)` (исходящее) в список `messages` чата.
  - Сохраняет конфиг.
- При любых исключениях (ошибка шифрования, отправки, поиска) метод молча завершается (ошибка подавляется).

## 🔗 Взаимодействие с другими модулями

### `api_helper`

Предоставляет низкоуровневые функции для работы с почтой:

- `receive(email, apikey, latest_uid)` → возвращает список `[(body, from_email), ...]` и новый `max_uid`.
- `send(email_from, email_to, apikey, encrypted_message)` → отправляет письмо с темой `"MESSAGE"`.

### `cryptohelper`

Содержит криптографические примитивы:

- `hash_password(password)` → SHA256 хеш (шестнадцатеричная строка).
- `encrypt(password, data)` / `decrypt(password, encrypted)` — симметричное шифрование мастер-паролем (через Fernet).
- `encrypt_message(message_key, route_key, chat_id, message)` → формирует пакет с route и сообщением, возвращает base64.
- `decrypt_message(route_key, message_key, packet_b64)` → расшифровывает сообщение.
- `validate_route(route_key, expected_chat_id, packet_b64)` → проверяет, что пакет предназначен для данного чата.

## ⚠️ Важные замечания по эксплуатации

1. **Мастер-пароль никогда не хранится** — только его хеш. Все операции, требующие пароля, сравнивают хеш переданного пароля с сохранённым.
2. **Ключи `routekey`, `route` и `enckey`** должны быть переданы собеседнику по защищённому каналу. Они не шифруются мастер-паролем при передаче (внутри чата `enckey` шифруется мастер-паролем, но сам `enckey` как строка передаётся открыто).
3. **Для работы с Яндекс.Почтой обязательно использовать пароль приложения**, а не обычный пароль.
4. Письма с темой **`"MESSAGE"`** используются как контейнеры; остальные письма игнорируются.
5. При длинных строках base64 почтовый сервер может разбивать их на части по 76 символов. Функции `api_helper` корректно восстанавливают исходную строку, поэтому данные не повреждаются.
6. Метод `receive` **не расшифровывает сообщения** — только проверяет маршрут и сохраняет зашифрованные пакеты. Расшифровка происходит в `get_messages_from_chat`.

## 🧪 Пример использования

```python
mgr = manager("config.json")

# Первоначальная настройка (только если hash пуст)
mgr.setup("my_master_password", "Alice")

# Добавляем свой почтовый ящик
mgr.setup_email("alice@yandex.ru", "yandex_app_password")

# Создаём чат с Бобом
enckey, routekey, route, my_name = mgr.create_chat("my_master_password", "Bob")
# Передаём Бобу: enckey, routekey, route, свой email "alice@yandex.ru"

# Отправляем сообщение
mgr.send_message("Bob", "Привет!", "bob@yandex.ru", "alice@yandex.ru", "my_master_password")

# Периодически проверяем новые письма
mgr.receive()
unread_chats = mgr.update()   # ['Bob']

# Читаем сообщения чата
messages, chat_name = mgr.get_messages_from_chat("Bob", "my_master_password")
print(messages)   # ['Привет!']
```
