# ОСНОВНЫЕ ЗАПРОСЫ И КРАТКОЕ ОПИСАНИЕ
## GET
### запросы страниц
* /home
* /workflow
* /
1. все они отправляют одну и ту же страницу, которая является клиентом
эта страницы реализует ПОЛНЫЙ функционал, то есть, пока не введен пароль, то она просит его ввести не показывая другого функционала
2. она может войти и тогда уже показывает другой функционал, если сервер еще не настроен(не было первично настройки), то он показывает окно регистрации
### запросы данных
* /update возвращает список чатов, чат айди каждого чата, количество непрочитанных сообщений в нем, основную информацию о профиле, принимает аргумент password, при запросе сравнивает хеши, совпадает возвращает 200, нет - 401
* /getchatmessages возвращает сообщения чата, принимает аргументы password, chatid, возвращает 200 в случае, если все данные совпадают(пароль проходит валидацию, chatid существует)  возвращает данные и код 200, в случае неправильног password, возвращает 401, в случае несуществующего chatid, возвращает 502

## POST
* /login
 принимает у себя только один аргумент password бекенд сравнивает хеши, если успешно, то возвращает 200, в случае если неверно, то возвращает 401, если аргумент не был передан возвращает 400
* /register регистрирует в себе пароль, работает только в том случае, если пароль не назначен, принимает единственный аргумент register, в случае если пароль не был назначен, и в запросе есть непустой аргумент password, возвращает 200, в случае если парол уже был назначен, возвращает 503, в случае если не был передан параметр password, либо он оказался пустым, возвращает 400
* /createchat генерирует новую чат, ключ шифрования, устанавливается название контакта, начинает ждать сообщение от пользователя, принмает password, name, в случае если password  не проходит валидацию возвращает 401, если  name нет или он пустой - 502, или нету какого-то аргумента, 400
* /sendmessage отправляет данные в чат, принимает password, chatid, message отпрвляет сообщение на указанный chatid, в случае несостыковки в данных, chatid не существет, message пустой возвращает 502, в случае проблем с паролем 401, в случае нехватки аргументов 400
* /mute перстаем получать сообщения от пользователя, принимает password, chatid, в случае проблем с паролем, 401, в случае проблем с chatid - 502, в случае нехватки аргументов - 400

## КОНФИГ
мы храним все данные в json
файл conf.json
### ЧТО ХРАНИТ КОНФИГ?
* hash - обычный числовой хеш
* chats - чаты

## ЧАТЫ
каждый чат это переписка с определенным пользователем
чат хранит следующее - 
1. ключ шифрования
2. список сообщений
3. почту, на которую нужно будет прислать ответ(заменяется с последним сообщением)
## СООБЩЕНИЕ
сообщение является словарем, которые хранит в себе
+ content - собственно сам текст сообщения
+ from - email того, кто отправил вам сообщение
+ nextkey - постфикс перед сообщением, чтобы мы могли определить, что нам пишут именно в этот чат
