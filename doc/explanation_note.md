# Реализованный функционал MUWL 

## Аутентификация и настройка
- Первичная настройка с установкой мастер-пароля (хеширование SHA256) и имени пользователя
- Вход по мастер-паролю с проверкой хеша
- Создание нового пароля

## Управление чатами
- Создание чата с генерацией уникальных криптоключей (routekey, route, enckey)
- Привязка email собеседника к чату
- Список всех чатов с отображением непрочитанных сообщений

## Отправка и получение сообщений
- Отправка текстовых сообщений через SMTP (Яндекс.Почта)
- Получение сообщений через IMAP с фильтрацией по теме "MESSAGE"
- Шифрование сообщений (AES-256-GCM) перед отправкой
- Автоматическая валидация маршрута и расшифровка входящих сообщений
- Сохранение истории сообщений в зашифрованном виде

## Медиафайлы
- Отправка файлов до 5 МБ (кодирование в base64)
- Автоматическое распознавание медиа-сообщений в чате
- Сохранение файлов на сервер и скачивание
- Просмотр изображений прямо в интерфейсе

## Управление почтовыми ящиками
- Добавление нескольких email-аккаунтов с API-ключами
- Инкрементальная загрузка сообщений (сохранение последнего UID)
- Автоматическая проверка всех ящиков при обновлении

## Экспорт и импорт
- Экспорт чата в JSON с полной историей сообщений
- Импорт переписки из JSON (в новый или существующий чат)
- Предпросмотр сообщений перед импортом
- Управление экспортированными файлами (список, скачивание, удаление)

## API эндпоинты (основные)
- `GET /update` – получение списка чатов и профиля
- `GET /getchatmessages` – получение сообщений чата
- `POST /login` – аутентификация
- `POST /register` – первичная настройка
- `POST /createchat` – создание чата
- `POST /sendmessage` – отправка текстового сообщения
- `POST /send_media` – отправка файла
- `POST /add_email` – добавление почтового ящика
- `POST /export_chat` – экспорт чата
- `POST /import_chat` – импорт чата

## Клиентский интерфейс
- Три режима: настройка, вход, основной (список чатов + активный чат)
- Автообновление чатов каждые 30 секунд
- Индикатор непрочитанных сообщений
- Отличие входящих от исходящих сообщений
- Модальные окна для всех действий
- Адаптивный дизайн (поддержка мобильных устройств)

## Хранение данных (conf.json)
- Хеш мастер-пароля
- Имя пользователя
- Список чатов (название, криптоключи, зашифрованные сообщения, email-адреса)
- Список email-аккаунтов (email, apikey, lastuid)

## Безопасность
- Мастер-пароль не хранится в открытом виде (только хеш)
- Каждый чат имеет уникальные ключи шифрования
- Ключи чатов шифруются мастер-паролем
- Валидация маршрута перед расшифровкой сообщений
- AES-256-GCM для шифрования сообщений (аутентификация через GCM tag)
